### Build wiring
- Root `build.gradle.kts`: declare `kotlin`/`compose` plugins with `apply false` so multiple
  modules can apply Compose without the `ConfigurationProblemReporterService` build-service clash.
- `ksp/build.gradle.kts`: pin Java 17 toolchain (was compiling under the daemon's JDK 21,
  producing a variant `:apps` couldn't consume).
- `settings.gradle.kts`: include `:catalog`, gated behind `-PexcludeCatalog`.
- `jitpack.yml`: `publishToMavenLocal -PexcludeCatalog` so the catalog's reference solutions are
  never compiled or published. Confirmed `:apps`/`:ksp` are the only published modules and
  nothing depends on `:catalog`, so solutions can't leak into the `mini-apps` artifact.
